之前一直没明白这种链条构造的原理，我们根据这个链条倒推回去，原理就很清晰了

首先需要知道的是，在2.7版本中，[].__class__.__bases__[0].__subclasses__()里面有好东西，里面好多库调用了我们需要的os模块，因此我了解的构造的一些利用链就是基于[].__class__.__bases__[0].__subclasses__()的

基础方法

__class__()方法

首先__class\()返回一个实例的类

__bases__方法

__bases__()返回一个父类

__subclasses__()方法

__subclases__()获取类的子类

那么这条基链的逻辑就清楚了，通过当前实例的类来访问其父类的所有子类，这个父类就是指的是object类
